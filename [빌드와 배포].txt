빌드(Build) : 소스코드를 실행 가능한 형태의 SW로 변환하는 작업

[빌드의 주요 단계]
1. 컴파일(Compile) : 고수준 언어의 소스 코드를 기계어로 번역

2. 링크(Link) : 개별적으로 컴파일된 후 링크 단계에서 하나로 묶어 실행 가능한 단일 프로그램(or 라이브러리)로 만듬

3. 테스트(Test) : 단위테스트, 통합테스트 등이 진행됨. 버그 발견 후 수정하는 과정 포함

[빌드 실행 시점]
1. 저장소(Git, SVN)에 PUSH할 때 (CI : 지속적통합)

2. 정기적으로 예약된 빌드 : CI/CD 파이프라인에서는 정기적으로 빌드를 스케줄하여 지속적으로 테스트

3. 특정 이벤트 발생 시 : 코드 저장소에 특정 태그가 추가될 때 or 특정 이슈가 해결될 때 빌드 실행될 수 있음

4. 수동으로 빌드를 실행할 때 : 테스트를 위해 -> 특정 개발 환경에서 테스트, 배포전 직접 빌드하여 SW 상태 확인을 위해

* 빌드가 실행되는 시점은 조직이나 프로젝트의 요구사항에 따라 다를 수 있으며, 자동화된 빌드 시스템을 구축하여 개발자들이 빠르고 쉽게 빌드를 실행할 수 있도록 하는 것이 좋다.

***

배포(Deployment) : 빌드 단계를 거쳐 생성된 실행 가능한 SW를 실제서버, 클라우드, 사용자의 기기에 배치
