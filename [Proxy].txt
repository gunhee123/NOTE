[Proxy]
어떤 다른 대상을 대신해서 행동하는 중개자 역할을 하는 개념.
컴퓨터 과학 및 네트워크 분야에서 프록시는 여러 가지 의미로 사용되며, 다양한 목적으로 활용된다.

1. 네트워크 프록시
 클라이언트와 서버 사이에서 중개자 역할을 수행하여 클라이언트의 요청을 대신 처리하거나 변형해서 서버로 전달하고, 서버의 응답을 클라이언트로 전달.
 네트워크 보안, 캐싱, 로드 밸런싱 등의 목적으로 사용.
 ex) 웹 프록시는 웹 브라우저와 웹 서버 사이에서 요청과 응답을 중개하는 역할을 수행

2. 보안 프록시
 보안 프록시는 네트워크 상의 클라이언트와 서버 사이에서 보안 기능을 수행. 내부 네트워크와 외부 네트워크 사이의 트래픽을 모닌터링하고 필터링하여 악성 코드나 해킹 시도 등을 차단하는 역할을 수행 

3. 캐시 프록시
 클라이언트가 서버로부터 리소스를 요청할 때, 해당 리소스의 사본을 저장하여 같은 요청이 반복될 때 클라이언트에게 사본을 제공하는 역할.
 이를 통해 네트워크 대역폭을 절약하고 응답 시간을 단축.

4. 리버스 프록시
 클라이언트와 서버 사이에서 요청을 받아 적절한 서버로 전달하는 역할.
 로드 밸런싱이나 보안 등의 목적으로 사용되며, 클라이언트는 실제 서버의 정보를 알 필요 없이 리버스 프록시에 요청을 보내면 된다.


* 프록시는 다양한 환경에서 사용되며, 보안, 성능 최적화, 익명성 보장 등의 목적으로 활용된다.
   네트워크 프록시나 웹 프록시 등은 일상적으로 웹 브라우징이나 네트워크 통신에서 사용되며, 이외에도 다양한 분야에서 활용
