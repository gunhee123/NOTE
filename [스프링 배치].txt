[스프링 배치]
대용량 데이터 처리를 위한 일괄처리 (Batch Processing) 프레임워크
대용량 데이터를 효율적으로 처리하기 위한 기능을 제공. 데이터베이스, 파일, 메시지 큐 등 다양한 소스에서 데이터를 읽어 처리하고 결과를 저장하는데 사용


<특징과 목적>
1. 대용량 데이터 처리

2. 트랜잭션 관리

3. 처리 과정 감시 및 관리 : 배치 작업의 실행 상태를 모니터링하고 로깅할 수 있는 기능을 제공.
                                       실패한 작업에 대한 롤백과 재시도 기능 제공

4. 병렬 처리 : 처리 속도를 향상시킬 수 있다.

5. 설정 가능한 스텝 : 배치 작업은 여러 단계로 나누어 구성 -> 각 단계는 데이터 읽기, 처리, 쓰기 등의 기능을 수행하며 이러한 단계는 설정 가능

6. 스킵 및 리트라이 기능 : 배치 작업 중 오류가 발생할 경우 스킵 또는 리트라이를 설정하여 오류를 처리하고 작업을 계속할 수 있다.

* 스프링 배치는 스프링 프레임워크의 일부로 제공되며, 개발자가 배치 작업을 간편하게 구성하고 관리할 수 있도록 도와준다.
* 주로 정기적으로 실행되는 데이터 처리 작업이나 ETL(Extract, Transform, Load) 작업, 대용량 데이터 마이그레이션 작업 등에 사용됩니다.
